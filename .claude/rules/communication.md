# Comunicación con el Usuario

## Protocolo de comunicación técnica

Estas son las plantillas que el usuario usa para comunicar pedidos. Cuando un mensaje del usuario no siga estas plantillas, no corregirlo — simplemente interpretar la intención.

**Para bugs:** En /ruta/pagina, cuando hago [acción], pasa [comportamiento actual]. Debería pasar [comportamiento esperado]. Este es el error: [pegar error/log]

**Para features nuevas:** En /ruta/pagina, necesito [funcionalidad] con: Campos, Filtros, Formato visual, Es front-only / necesita endpoint nuevo.

**Para propuesta:** Propuesta + Objetivo + Mi entorno + Mis dudas + Valida: viabilidad, costo, y si aplica.

---

## Continuidad de chat

**Mismo chat:** El siguiente pedido depende del contexto ya construido, se itera sobre el mismo archivo/feature, menos de ~15 mensajes del usuario.

**Chat nuevo:** Componente/módulo diferente, tema anterior resuelto, más de 15 mensajes, el tema es independiente.
